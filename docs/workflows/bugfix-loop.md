# 버그 수정 루프 (재현 → 수정 → 검증)

## 1) 재현

- 증상을 “관찰 가능”하게 만든다 (에러 메시지, 로그, 최소 재현 절차)
- 재현 절차를 PR 본문 또는 관련 문서에 남긴다

## 2) 원인 규명

- 경계(입력/외부 I/O)에서 형태를 확정한다 (검증/파싱)
- “추측에 의존하는 형태”를 코드에 퍼뜨리지 않는다

## 3) 수정

- 프로젝트 규칙: 에러 처리는 **try-catch**로 구현한다
- 영향 범위를 작게 유지한다 (최소 수정)

## 4) 검증

- 같은 재현 절차로 “수정 전/후”를 비교한다
- 가능하면 테스트(또는 최소 회귀 체크)를 추가한다

## 5) 재발 방지(가드레일)

- 같은 유형이 다시 생기면 문서가 아니라 **테스트/린트/CI**로 막는다
