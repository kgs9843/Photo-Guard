# 제품 / 도메인 (SoT)

## 한 줄

**Photo Guard**는 브라우저 안에서 사진을 고르고, 메타데이터를 **검사한 뒤 제거**한 뒤, 더 안전하게 **공유·저장**할 수 있게 돕는 웹앱이다. 원본은 서버로 보내지 않는다.

## 핵심 사용자 여정

1. **홈(대시보드)** — 사진 선택(최대 10장) → Clean 화면으로 이동
2. **Clean** — EXIF 등 메타 요약 확인 → 정제(Cleaning)로 이동
3. **Cleaning** — 정제 진행 → 완료 후 다운로드 또는 공유
4. **기록** — 정제에 성공한 항목을 IndexedDB에 남기고, 목록·상세에서 “무엇이 지워졌는지” 확인
5. **설정** — 언어(한국어/English), 보내기 형식(JPEG/PNG), 로컬 기록 삭제

부가: **개인정보 처리방침**, **오픈소스 라이선스**, **오프라인** 안내 화면.

## 용어

| 용어            | 의미                                                                 |
| --------------- | -------------------------------------------------------------------- |
| 정제 / Cleaning | 캔버스 재인코딩으로 **새 이미지 blob**을 만들고 메타를 제거하는 과정 |
| 기록            | 정제 성공 시 IndexedDB에 저장되는 **썸네일 + 요약 + blob**           |
| 추출 형식       | 설정의 JPEG/PNG 선택 → 다운로드·정제 출력 MIME에 반영                |

## 수용 기준(요약)

- **Clean**: 메타데이터가 없으면(요약 기준) 정제로 보내는 CTA는 비활성·회색 처리
- **Cleaning**: 정제 실패 시 사용자에게 실패 개수를 알린다. 성공 시 기록 저장이 가능하면 IndexedDB에 남기고 목록을 갱신한다
- **기록 상세**: 제거 전 탐지 요약 + 기타 EXIF 태그 상세(저장된 범위 내)를 볼 수 있다
- **설정**: 로컬 기록 삭제 시 IndexedDB 기록은 지우되, 추출 형식 설정은 유지할 수 있다

## 외부 지식

노션·슬랙 등 **레포 밖에만 있는 제품 결정은 없는 것으로 취급**한다. 바뀌면 이 파일 또는 `docs/exec-plans`에 반영한다.
